## Пояснение по реализации ввода и нормализации:
Во время нормализации поля, оно будет неинтерактивным до окончания всех падений и анимаций.
Такой подход более надёжный и предсказуемый.

Разрешение частичных свайпов в этот момент порождает много спорных ситуаций
(одновременные падения, пересечения с вводом игрока, конфликт состояний),
которые требуют сильно более сложной логики и легко приводят к багам. Галочку в насройках для включения интерактивности можно поставить, 
но работать будет с багами в случае множественных рандомных свайпов Элементов подряд.

Для тестового задания я сознательно выбрал стабильное и безопасное решение.
При необходимости это поведение можно расширить в будущем.
